OBS Dual — Horizontal & Vertical

Controlá dos instancias de OBS con distintos perfiles y sincronización de escenas en tiempo real

🎬
OBS Dual — Dos OBS al mismo tiempo, un solo control

Lanzá un OBS con perfil Horizontal (16:9) y otro con perfil Vertical (9:16) simultáneamente. Cuando cambiás de escena en uno, el otro cambia automáticamente. Desde la PC Gaming podés controlar las escenas de la PC Streaming en tiempo real, con atajos de teclado globales.

📌 Resumen rápido — ¿Qué se configura dónde?

Antes de entrar en detalles, esta es la respuesta directa a la pregunta más frecuente:

🖥️ SE CONFIGURA EN OBS
  • ✔ Crear perfil Horizontal (resolución, stream key)
  • ✔ Crear perfil Vertical (resolución, stream key)
  • ✔ Crear colección de escenas Horizontal
  • ✔ Crear colección de escenas Vertical
  • ✔ Activar WebSocket en perfil Horizontal → puerto 4455
  • ✔ Activar WebSocket en perfil Vertical → puerto 4456
  • ✘ Nada más — ningún plugin extra requerido
🎛️ SE CONFIGURA EN ESTA APP
  • ✔ Dirección WebSocket de cada OBS (ws://…:4455)
  • ✔ Contraseña WebSocket (si la configuraste)
  • ✔ Nombre del perfil y colección para cada instancia
  • ✔ Mapeo de escenas H ↔ V
  • ✔ Ruta al ejecutable (si no está en la ruta default)
  • ✔ Room ID y modo para el control remoto
  • ✔ Atajos de teclado globales (PC Gaming)
🎮 Control Remoto — 100% en la app, cero en OBS
El control desde la PC Gaming funciona a través de Supabase Realtime (ya configurado en la app). No hay que instalar nada extra en la PC de streaming ni en OBS. Solo abrís la app en ambas PCs, configurás el mismo Room ID, elegís el modo de cada PC y listo.
📋 ¿Cómo funciona? — Flujo completo
1
Dos perfiles, dos colecciones — todo en OBS
Configurás un perfil Horizontal (1920×1080, con tu stream key de Kick/YouTube) y un perfil Vertical (1080×1920, con tu stream key de TikTok/Kick vertical). Cada perfil tiene su propio WebSocket en un puerto distinto. Las colecciones de escenas son independientes por formato.
2
La app lanza y controla ambos OBS a la vez
Desde esta app hacés click en "Lanzar ambos OBS": se abren dos procesos OBS en paralelo, cada uno con su perfil y colección. Luego conectás via WebSocket y la app tiene control total de ambas instancias: escenas, stream, grabación.
3
Mapeo de escenas = sincronización automática
Definís qué escena del OBS Vertical corresponde a cada escena del OBS Horizontal. Cuando cambiás en uno (desde la app o desde OBS directamente), el otro cambia automáticamente a su escena mapeada. El sistema previene loops de sincronización con un cooldown de 300ms.
4
Control remoto desde la PC Gaming via Supabase Realtime
Abrís esta app en la PC Gaming en modo "PC Gaming" y en la PC Streaming en modo "PC Streaming". Ambas usan el mismo Room ID para comunicarse. La PC Gaming ve las escenas disponibles en tiempo real y puede cambiarlas con un click o con atajos de teclado globales. La latencia es menor a 100ms via internet.
✅ Requisitos previos
  • 🖥️ OBS Studio 28 o superior en la PC de streaming — incluye WebSocket de serie, sin plugins. Descargar OBS
  • 💾 16 GB RAM mínimo recomendados para correr dos instancias simultáneas, más el juego. Usar NVENC / AMF / QuickSync en ambos perfiles para que el encoding lo maneje la GPU.
  • 🌐 Buena conexión a internet si streaméas en ambas al mismo tiempo. Dos streams de ~6000 kbps cada uno = ~12 Mbps de upload.
  • 🔑 Dos stream keys distintas — una para el canal horizontal y otra para el vertical (pueden ser dos cuentas, dos plataformas, o funciones multistream).
  • ☁️ Supabase configurado en la app (ya requerido por otras funciones) — necesario para el control remoto entre PCs. Sin Supabase el control local sigue funcionando.
  • 🎮 Esta app instalada en la PC Gaming también — solo si vas a usar el control remoto.
🖥️ Configuración en OBS — Paso a paso completo

Solo hacés esto una vez. Una vez que los perfiles y el WebSocket están configurados, no volvés a tocar OBS para nada relacionado con OBS Dual ni con el control remoto.

Fase 1 — Crear los perfiles

1 Abrí OBS → menú superior Perfil → Nuevo → escribí Horizontal → OK
2 Con el perfil Horizontal activo, abrí Configuración (engranaje):
• Video → Resolución base: 1920×1080, Resolución de salida: 1920×1080, FPS: 60
• Stream → Servicio: Kick / YouTube → pegá tu stream key horizontal
• Salida → Encoding: NVENC H.264 (o AMF/QuickSync según tu GPU)
3 Menú Perfil → Nuevo → escribí Vertical → OK
4 Con el perfil Vertical activo, abrí Configuración:
• Video → Resolución base: 1080×1920, Resolución de salida: 1080×1920
• Stream → Servicio y stream key del canal vertical (TikTok, Kick, YouTube Shorts, etc.)
• Salida → Encoding: NVENC H.264 (puede ser la misma GPU, distintas instancias del encoder)

Fase 2 — Crear las colecciones de escenas

1 Menú Colección de escenas → Nueva → nombrala Horizontal
2 Creá tus escenas en formato 16:9: En Vivo, BRB, Inicio, Fin Stream, Solo Cam, etc.
Agregá las fuentes (captura de juego, cámara, alertas) posicionadas para 1920×1080.
3 Menú Colección de escenas → Nueva → nombrala Vertical
4 Creá las mismas escenas adaptadas al formato 9:16. Truco clave: usá exactamente los mismos nombres de escenas que en la colección Horizontal — así el Auto-mapeo de esta app las vincula automáticamente sin configuración manual.
5 Para adaptar las fuentes: en cada escena, reposicioná la captura de juego para que ocupe el área central vertical. Podés usar el plugin Source Clone para reutilizar fuentes entre colecciones.

Fase 3 — Activar el servidor WebSocket en cada perfil

El WebSocket es la única configuración técnica necesaria en OBS. Es lo que permite que esta app se comunique con OBS.

1 Con el perfil Horizontal activo:
Menú Herramientas → Configuración del servidor WebSocket
✓ Tildar "Activar servidor WebSocket"
• Puerto del servidor: 4455
• Autenticación: opcional pero recomendada. Si la activás, anotá la contraseña para configurarla en esta app.
→ Aplicar / OK
2 Cambiá al perfil Vertical: menú Perfil → Vertical
Volvé a ir a Herramientas → Configuración del servidor WebSocket
✓ Tildar "Activar servidor WebSocket"
• Puerto del servidor: 4456 — distinto al horizontal, obligatorio
• Autenticación: la misma contraseña o una diferente (podés usar la misma para simplificar)
→ Aplicar / OK
3 Importante: el WebSocket se guarda por perfil. Al lanzar OBS con --profile Horizontal usará el puerto 4455 automáticamente. Con --profile Vertical usará el 4456. No tenés que cambiarlo a mano cada vez.

💡 Verificación rápida: Con OBS abierto, abrí CMD y ejecutá netstat -ano | findstr :4455 — si aparece una línea con LISTENING, el WebSocket está activo y escuchando.

💡 Tip de performance: Activá NVENC/AMF/QuickSync en ambos perfiles. Dos encoders de GPU en paralelo en la misma placa es perfectamente viable en GPUs modernas (RTX 3000+ soporta hasta 5 sesiones NVENC simultáneas).

🎛️ Configuración en la App — Paso a paso completo

Fase 1 — Ajustes de conexión WebSocket

1 En el panel OBS Horizontal de esta pantalla, expandí los Ajustes de conexión:
• Dirección: ws://127.0.0.1:4455 (si OBS está en la misma PC)
• Contraseña: la que configuraste en OBS, o vacío si no usás autenticación
• Perfil OBS: Horizontal (exactamente como aparece en el menú Perfil de OBS)
• Colección: Horizontal (exactamente como aparece en el menú Colección de escenas)
2 Lo mismo para el panel OBS Vertical:
• Dirección: ws://127.0.0.1:4456
• Contraseña: la que configuraste
• Perfil OBS: Vertical
• Colección: Vertical
3 Hacé click en "Guardar config" en cada panel — los datos quedan guardados para la próxima vez que abras la app.

Fase 2 — Lanzar y conectar

1 Click en "Lanzar ambos OBS" en la barra de arriba — abre dos procesos OBS en paralelo, cada uno con su perfil y colección definidos en los ajustes.
2 Esperá 3-5 segundos a que OBS cargue completamente (hasta que veas las ventanas de OBS en la barra de tareas).
3 Click en "Conectar ambos" — la app establece la conexión WebSocket con cada instancia. Los indicadores de estado en cada panel deben ponerse en verde.
4 Si un panel no conecta, verificá que OBS esté abierto, el WebSocket esté activo, y la dirección/contraseña sean correctas.

Fase 3 — Configurar el mapeo de escenas

1 Una vez conectado, en el panel Mapeo de escenas click en "Auto-mapear". Si los nombres de escenas son iguales en ambas colecciones, se vincularán solos.
2 Para escenas con nombres distintos, usá los selectores de la tabla para vincularlas manualmente (ej: "Gaming 16:9" ↔ "Gaming vertical").
3 Click en "Guardar mapa" — el mapa queda persistido y se recarga automáticamente en la próxima sesión.
4 Activá el toggle "Sync escenas" en la barra de control. A partir de ahora, cambiar escena en cualquier OBS (desde aquí o desde OBS directamente) sincroniza el otro automáticamente.
🎮 Control Remoto entre PCs — Configuración detallada

No se toca OBS para nada en este paso. El control remoto es comunicación directa entre las dos instancias de la app via Supabase Realtime. La PC Streaming recibe los comandos y los reenvía a OBS via WebSocket local.

¿Cómo funciona internamente?

G
PC Gaming (modo Gaming) — tenés la app abierta mientras jugás. Presionás un atajo de teclado o clickeás una escena → la app envía un comando de escena al canal de Supabase Realtime.
S
PC Streaming (modo Streaming) — tiene la app abierta con ambos OBS conectados. Recibe el comando de Supabase y lo ejecuta en OBS via WebSocket local con latencia <100ms.

Configuración en la PC Streaming

1 Abrí esta app en la PC Streaming y andá a la pestaña OBS Dual
2 En la sección Control Remoto, seleccioná el modo 📺 PC Streaming
3 En el campo Room ID, escribí un identificador único para tu sala — puede ser tu nombre de canal, un código, lo que quieras. Por ejemplo: micanal_obs. Tiene que ser exactamente igual en ambas PCs.
4 Click en Guardar y luego en Activar. La PC Streaming queda escuchando comandos. Cuando un comando llega, cambia la escena en los OBS conectados y responde con el estado actualizado.
5 Asegurate de que los dos OBS estén conectados antes de activar el control remoto, o activá el auto-conectar al iniciar en los ajustes para que todo se conecte solo al abrir la app.

Configuración en la PC Gaming

1 Abrí esta app en la PC Gaming y andá a la pestaña OBS Dual
2 En la sección Control Remoto, seleccioná el modo 🎮 PC Gaming
3 Ingresá el mismo Room ID que configuraste en la PC Streaming
4 Click en Guardar y luego en Activar. Aparecerá la grilla de escenas disponibles (llegan de la PC Streaming automáticamente). Si no aparecen, click en "Pedir estado".
5 (Opcional) Configurá los Atajos de teclado globales: elegí una escena, el OBS destino (Horizontal, Vertical, o Ambos) y grabá el combo de teclas. Los atajos funcionan incluso con el juego en primer plano — son atajos de sistema globales. Usá combos con al menos un modificador (Ctrl/Alt/Shift) para evitar conflictos con el juego.
6 Click en "Guardar y aplicar" para registrar los atajos. El estado (✓ / ✗) te indica si el atajo fue registrado con éxito o si ya está en uso por otra app.

Notas importantes

  • 🌐 Ambas PCs necesitan conexión a internet para el canal Realtime de Supabase
  • ⚡ La latencia típica es menor a 100ms — prácticamente instantáneo para cambios de escena
  • 🔄 Si se corta la conexión, la app reconecta automáticamente con backoff exponencial
  • 💓 El indicador de heartbeat en la PC Gaming muestra cuándo fue la última señal de la PC Streaming, para saber si está viva
  • 🎮 Los atajos de teclado son globales del sistema operativo — funcionan desde cualquier app, incluso con el juego en pantalla completa
  • ⚠️ Algunos combos pueden estar reservados por Windows o el juego. Si ✗ aparece en el estado del atajo, probá con otro combo
  • 📡 El Room ID es un identificador público en el canal de Supabase. Usá algo único como tunombre_obs2024 para evitar colisiones con otros usuarios de la app
⚠️ Puertos WebSocket — Conflictos y solución

Cada instancia de OBS debe usar un puerto WebSocket distinto. Si dos instancias intentan usar el mismo puerto, la segunda no podrá iniciar el servidor y esta app no podrá conectarse.

→ Puerto por defecto de OBS: 4455 — asignalo al OBS Horizontal
→ OBS Vertical: cambiarlo a 4456 en Herramientas → Configuración del servidor WebSocket
→ Si Audio Link también usa un OBS separado en el puerto 4455, usá 4457 para el OBS Vertical y actualizá la dirección en los ajustes de esta sección
→ Para verificar si un puerto está en uso en Windows:
netstat -ano | findstr :4455 — si aparece LISTENING, el puerto está ocupado

💡 Los puertos se guardan por perfil de OBS. Al usar --profile Horizontal y --profile Vertical para lanzar cada instancia, cada una carga su propio WebSocket config automáticamente. No hay conflicto si configuraste los puertos correctamente en cada perfil.

🔀 ¿Cómo funciona el mapeo de escenas?

El mapeo define qué escena del OBS Vertical se activa cuando cambiás una escena en el OBS Horizontal (y viceversa). Es bidireccional.

1 Auto-mapeo: si tus escenas tienen el mismo nombre en ambas colecciones, click en "Auto-mapear" y el sistema las vincula solas. Es la forma más rápida.
2 Mapeo manual: si los nombres difieren (ej: "En Vivo" ↔ "En Vivo Vertical"), usá los selectores del panel de Mapeo para vincularlas una a una.
3 Guardá el mapa — queda persistido y se aplica automáticamente cada vez que abrís la app.
4 Sync bidireccional: cambiando desde OBS Horizontal → cambia Vertical; cambiando desde OBS Vertical → cambia Horizontal. También aplica cuando el cambio viene desde un atajo de la PC Gaming.
5 Anti-loop: el sistema tiene un cooldown de 300ms para evitar que el cambio en uno dispare el cambio en el otro, que dispare de vuelta el primero, y así infinitamente.

💡 Podés desactivar el sync temporalmente con el toggle "Sync escenas" en la barra de control sin perder el mapa guardado. Útil si querés controlar cada OBS de forma independiente por un momento.

📂 Ruta al ejecutable de OBS — ¿cuándo configurarla?

La app detecta OBS automáticamente en las rutas más comunes de Windows. Solo necesitás configurar la ruta manualmente si:

→ Instalaste OBS en una carpeta personalizada (no la carpeta default de Program Files)
→ Tenés múltiples versiones de OBS instaladas y querés usar una versión específica para cada instancia
→ Usás una build portable de OBS (descargada como ZIP, sin instalador)

Ruta típica de instalación estándar:
C:\Program Files\obs-studio\bin\64bit\obs64.exe

💡 Para encontrarla rápido: click derecho en el ícono de OBS → Abrir ubicación del archivo → copiá la ruta completa incluyendo el .exe.
